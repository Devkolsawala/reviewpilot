## Prompt engineering basics for review replies

Before the templates, here's what every Play Store review reply prompt needs in 2026:

1. **Role assignment.** "You are the founder of [app name]" or "You are the support lead of [app name]." Anchors the model's voice.
2. **Product context.** 2–3 sentences on what your app does, who uses it, and your brand tone. Without this, GPT writes generic SaaS-speak.
3. **Hard constraint: 350 characters max, including spaces.** Specify it twice — once at the start, once near the end. GPT-4 often violates length constraints on the first pass.
4. **Tone target.** "Warm but professional." "Direct but empathetic." "Friendly Indian English with no slang." Specifying tone explicitly prevents drift.
5. **Output format.** "Output the reply only, no preamble." Stops GPT from prefacing with "Here's a reply that..."
6. **One worked example.** Show one before/after pair. The model massively improves with a single example versus zero.

Most prompts in this library follow this structure. Adapt the placeholders to your app.

## 5-star thank you prompts

**Prompt 1.1 — Short and warm**

```
You are the founder of {app_name}, an Android app that helps {user_persona} {core_value}. Your tone is warm but never saccharine. A user left this 5-star review: "{review_text}"

Write a reply under 350 characters that:
- Thanks them by name if visible
- References one specific thing they mentioned
- Optionally teases an upcoming feature
- Sounds human, not corporate

Output the reply only, no preamble. Stay under 350 chars including spaces.
```

**Worked example:**
- Inputs: `app_name = "Notesly"`, `user_persona = "writers and researchers"`, `core_value = "organize thoughts across devices"`, review = "Switched from Notion, love the offline mode."

GPT-4 output: "Welcome aboard from Notion-land. Offline mode was a six-month build, so it means a lot to hear it's actually useful. Cross-device markdown lands in v5 next month." (172 chars)

**Prompt 1.2 — With a referral nudge**

```
You are the support lead of {app_name}. A user left a 5-star review: "{review_text}"

Write a 350-character reply that:
- Thanks them sincerely
- Mentions one specific feature they cited
- Includes a soft, single-sentence referral ask
- Avoids any "valued customer" language

Output reply only, under 350 chars.
```

**Prompt 1.3 — For power users**

```
You are the founder of {app_name}. A clearly invested user left this 5-star review: "{review_text}"

They've used the app long enough to be a power user. Write a reply that:
- Acknowledges their depth of usage
- Asks one open-ended question about how they'd improve the app
- Stays under 350 chars
- Sounds like a peer, not support

Output reply only.
```

This is the highest-ROI prompt in the set. Power users who reply to your question give you the best feature ideas you'll get all quarter.

## 1-star bug report prompts

**Prompt 2.1 — Bug acknowledgement with a fix path**

```
You are the founder of {app_name}. A user left this 1-star review describing a bug: "{review_text}"

Known relevant bugs:
- {bug_1}
- {bug_2}

Write a 350-character reply that:
- Apologizes briefly (one sentence)
- Identifies which known bug they likely hit (or asks if you can't tell)
- Provides a specific fix path or workaround
- Closes with an email for follow-up
- Avoids defensive language

Output reply only, under 350 chars.
```

**Worked example:**
- Inputs: `app_name = "BudgetPay"`, bug_1 = "OnePlus battery optimization kills sync", bug_2 = "Payment double-charge on Razorpay", review = "App lost all my data overnight."

GPT-4 output: "Sorry — data loss is unacceptable. This sounds like the OnePlus battery-optimization bug that kills background sync. Settings → Battery → App management → BudgetPay → Don't optimize. Email support@budgetpay.app with your user ID if data isn't back." (252 chars)

**Prompt 2.2 — Device-specific issue**

```
You are the founder of {app_name}. A 1-star reviewer reported a device-specific issue: "{review_text}"

Their likely device class: {device_class}

Write a 350-char reply that:
- Acknowledges the issue without making excuses
- Notes the device-specific cause concisely
- Gives the fix steps in one short sentence
- Confirms timeline if a real fix is shipping

Output reply only.
```

**Prompt 2.3 — Crash with no detail**

```
You are the founder of {app_name}. A user left a 1-star review saying only "{review_text}" with no useful detail.

Write a reply that:
- Apologizes briefly
- Asks for device model and Android version in a friendly way
- Provides a direct support email
- Does NOT sound like a form letter

Stay under 350 characters. Output reply only.
```

## Feature request prompts

**Prompt 3.1 — Acknowledge and set expectations**

```
You are the product lead of {app_name}. A user requested a feature: "{review_text}"

Roadmap status of this feature: {roadmap_status}

Write a 350-char reply that:
- Validates their request specifically
- States the roadmap status honestly (in dev / under consideration / not planned)
- Offers a workaround if the feature isn't coming
- Closes with a beta sign-up link if it is coming

Output reply only.
```

**Prompt 3.2 — Feature already exists**

```
You are the support lead of {app_name}. A reviewer requested a feature that already exists: "{review_text}"

The feature is accessed via: {feature_path}

Write a 350-char reply that:
- Doesn't make the user feel dumb
- Names the feature clearly
- Provides the exact menu path
- Includes the app version where it shipped (in case they're outdated)

Output reply only.
```

## Angry rant de-escalation prompts

**Prompt 4.1 — Angry, vague rant**

```
You are the founder of {app_name}. A user left this hostile 1-star review: "{review_text}"

The review has no specific complaint, only anger. Write a 350-char reply that:
- Does NOT match their tone
- Does NOT apologize for things you don't understand
- Asks one specific question to understand what went wrong
- Offers a direct email for follow-up
- Sounds like a human, not a script

Output reply only.
```

**Prompt 4.2 — Justified anger with specifics**

```
You are the founder of {app_name}. A user is justifiably angry: "{review_text}"

Their complaint is specific and valid. Write a 350-char reply that:
- Acknowledges they're right
- Apologizes specifically for what went wrong
- Provides a concrete fix or refund path
- Names yourself by first name so it sounds personal
- Includes an email

Output reply only, under 350 chars.
```

## Spam and competitor-planted reviews

**Prompt 6.1 — Spam / competitor mention**

```
You are the support lead of {app_name}. A review appears to be spam or recommending a competitor: "{review_text}"

Write a 350-char reply addressed to OTHER READERS, not the reviewer. Goal:
- Neutralize the spam without engaging emotionally
- Mention you've reported the review to Google
- Briefly point real users to your support channel
- Do NOT accuse the reviewer of being a competitor

Output reply only.
```

This prompt is the single most-used in the spam category because it threads the needle — protects your rating without escalating.

## Why generic ChatGPT replies hurt your rating

The risk with ChatGPT replies is consistency, not capability. The model produces similar phrasings repeatedly when given similar prompts. Run a batch of 20 1-star reviews through the same prompt and you'll see "We're sorry to hear..." appear in 14 of them.

This matters because:

1. **Google's spam detection flags templated replies.** Identical or near-identical text across your replies in a short window can trip the algorithm.
2. **Users notice.** Users scrolling Play Store reviews see the same opener five times and immediately recognize templated AI. The trust signal collapses.
3. **The 350-char limit is binding.** GPT routinely overshoots, especially on multi-language outputs. Manual trimming on every reply burns time.

The fix is one of three:

- **Personalize the first line manually.** Before sending any AI reply, rewrite the opening sentence using something specific from the review (a feature name, the user's name, a device model). This single edit defeats spam detection and changes the reading experience.
- **Use a purpose-built tool.** Tools like ReviewPilot wrap an LLM with built-in character counting, context injection, brand-voice memory, and a sentiment classifier. The output stays under 350 chars automatically, varies naturally across reviews, and ships via the Play Console API without copy-paste. The [Play Store Character Counter](/tools/play-store-character-counter) handles the simplest version of this for free.
- **Rotate prompts.** If you stick with ChatGPT manually, keep 4–5 versions of each prompt and rotate. Reduces phrasing repetition.

## When ChatGPT is the right answer

For low volume — say, under 30 reviews per week — ChatGPT plus manual copy-paste is fine. You're not yet at the scale where workflow overhead matters, and the cost of a $20/mo subscription beats a $50/mo dedicated tool.

For higher volume — 100+ reviews per week — the workflow tax compounds. Each reply takes 2–3 minutes (paste review, run prompt, check length, paste back into Play Console). At 100 replies a week, that's 3–5 hours of pure context switching. A purpose-built tool kills that.
